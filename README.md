# profcruz.com — Personal Blog

A lightweight static blog. No frameworks, no dependencies, no database.  
Write in Markdown → run build script → deploy.

---

## Structure

```
profcruz-blog/
├── index.html          ← Blog home (auto-generated by build.py)
├── about.html          ← About page (edit directly)
├── build.py            ← Converts drafts → posts, rebuilds index
├── assets/
│   └── style.css       ← All styles
├── drafts/             ← Write new posts here as .md files
│   └── my-post.md
├── posts/              ← Generated HTML posts (don't edit directly)
│   └── my-post.html
└── README.md
```

---

## Writing a New Post

1. Create a `.md` file in `/drafts`:

```markdown
---
title: Your Post Title
subtitle: Optional subtitle shown below the title
date: March 1, 2026
tags: Strategy, Governance
excerpt: A 1-2 sentence summary shown on the index page.
---

Your post content here. Standard Markdown:
**bold**, *italic*, [links](https://example.com),
## headings, > blockquotes, - lists, etc.
```

2. Run the build:

```bash
python build.py
```

3. The script generates `posts/your-post-title.html` and updates `index.html`.

4. Commit and push to GitHub → Cloudflare auto-deploys.

## Deployment (One-Time Setup)

### Step 1: Create a GitHub Repository

1. Go to [github.com/new](https://github.com/new)
2. Name it `profcruz-blog` (or whatever you prefer)
3. Set to **Public** or **Private** (both work)
4. Push this folder:

```bash
cd profcruz-blog
git init
git add .
git commit -m "Initial blog setup"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/profcruz-blog.git
git push -u origin main
```

### Step 2: Set Up Cloudflare Pages (Free)

1. Go to [dash.cloudflare.com](https://dash.cloudflare.com) → Sign up or log in
2. Click **Workers & Pages** → **Create** → **Pages** → **Connect to Git**
3. Authorize GitHub and select your `profcruz-blog` repository
4. Configuration:
   - **Production branch**: `main`
   - **Build command**: *(leave empty)*
   - **Build output directory**: `/` (or just leave default)
5. Click **Save and Deploy**

### Step 3: Connect Your Domain

1. In Cloudflare Pages, go to your project → **Custom domains**
2. Add `profcruz.com`
3. Cloudflare will guide you to point your domain's nameservers to Cloudflare  
   (or add a CNAME record if your registrar supports it)
4. SSL is automatic. Done.

### Publishing Workflow (Ongoing)

```bash
# Write your post
nano drafts/my-new-post.md

# Build
python build.py

# Deploy
git add .
git commit -m "New post: My New Post"
git push
```

Cloudflare auto-deploys on push. Live in ~30 seconds.

## Customization

- **Colors/fonts**: Edit CSS variables at the top of `assets/style.css`
- **Site name/tagline**: Edit the hero section in `build.py` → `generate_index()`
- **Footer links**: Update in `build.py` templates and in `about.html`
- **Add pages**: Create new `.html` files, add nav links in the header

## No Dependencies Required

The build script uses only Python standard library. Python 3.8+ is all you need.  
No npm, no node, no Ruby, no Hugo, no Jekyll, no Gatsby.
